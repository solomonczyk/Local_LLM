# UX/UI guide (local)

## 1) Core UX principles
- User-centered design
- Consistency across interface
- Clear visual hierarchy
- Immediate feedback on actions
- Error prevention over error messages

## 2) Usability heuristics
- Visibility of system status
- Match between system and real world
- User control and freedom (undo/redo)
- Consistency and standards
- Help users recognize, diagnose, and recover from errors

## 3) Interface design
- Clear call-to-action buttons
- Adequate spacing and whitespace
- Readable typography (16px+ for body text)
- Color contrast for accessibility (WCAG AA minimum)
- Touch targets 44x44px minimum for mobile

## 4) User flow optimization
- Minimize steps to complete tasks
- Progressive disclosure (show what's needed when needed)
- Clear navigation and breadcrumbs
- Logical grouping of related items
- Helpful empty states

## 5) Common UX issues
- Too many choices (decision paralysis)
- Hidden navigation
- Unclear error messages
- Inconsistent interactions
- No loading indicators

## 6) Accessibility requirements
- Keyboard navigation support
- Screen reader compatibility
- Focus indicators
- Skip links for navigation
- Captions for media

## 7) Output rules for UX agent
Always return:
- UX issues (with severity)
- User pain points identified
- Improvement recommendations
- Accessibility concerns
- Quick wins for better UX
